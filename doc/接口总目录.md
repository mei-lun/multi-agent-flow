# Multi Agent Flow 接口总目录

> 本文是接口导航。字段定义与逐步逻辑以对应 `schemas.py`、`service.py`、`router.py` 和目录 `README.md` 为准。接口均为规范，没有业务实现。

## 1. 公共 REST API

### 1.1 IAM 与系统（8 个操作）

| 方法与路径 | 输入 | 输出 | Service 处理要点 |
|---|---|---|---|
| `POST /api/v1/auth/login` | `LoginRequest`：用户名、密码 | `SessionView` | 用户状态、恒定时间密码校验、创建会话；错误不泄露用户是否存在 |
| `POST /api/v1/auth/logout` | session header | 无正文 | 幂等撤销当前 session，保留审计 |
| `GET /api/v1/me` | `ActorContext` | `UserView` | 从服务端重新计算权限 |
| `GET /api/v1/users` | `UserQuery` | `UserPage` | 管理权限、过滤、游标分页 |
| `POST /api/v1/users` | `CreateUserRequest` | `UserView` | 用户名唯一、密码哈希、权限键、幂等 |
| `PATCH /api/v1/users/{id}` | `UpdateUserRequest` | `UserView` | 乐观锁；不能禁用最后一个管理员；禁用后撤销会话 |
| `GET /api/v1/settings/{key}` | path key | `SettingView` | 仅允许预定义设置；Secret 脱敏 |
| `PUT /api/v1/settings/{key}` | `PutSettingRequest` | `SettingView` | Schema、权限、版本；敏感值转 SecretStore |

代码位置：`modules/iam/`。

## 2. Git 跨节点接口

跨机器不提供自建 `/internal/v1` REST API。接口由受保护分支、文件 Schema 和 Git 提交组成。

| Git 接口 | 写入者 | 输入 | 输出与关键逻辑 |
|---|---|---|---|
| `maf/control:.maf/tasks/<task-id>.yaml` | 中央 Scheduler | `CoordinationTask` | 权威任务；单写、Schema、状态转换、expected head |
| `maf/control:.maf/nodes/<node-id>.yaml` | 中央 Scheduler | 已接受 Node Event | 权威节点能力和状态 |
| `maf/control:.maf/events/<event-id>.json` | 中央 Scheduler | `EventDecision` | 已接受/拒绝事件的可重放记录 |
| `maf/control:.maf/status.md` | 中央 Scheduler | tasks/nodes/events | 自动生成的人类总览，禁止手改 |
| `maf/node/<node-id>:.maf/events/<event-id>.json` | 对应节点 | `CoordinationEvent` | Claim、Progress、Blocked、Submission、Abandon 申请 |
| `maf/task/<task-id>/e<epoch>-<node-id>` | 当前被分配节点 | 代码、文档、测试、脱敏报告 | 候选交付；base/owner/epoch 必须与 control 一致 |

核心 Git 接口代码位于 `modules/git_coordination/`、`apps/runner/git_client.py` 和
`maf_contracts/coordination.py`。中央确认 control commit 后再更新 SQLite 投影。

## 4. 查阅顺序

实现一个接口时按以下顺序阅读：

1. 本目录找到 HTTP 或非 HTTP 接口；
2. 打开对应 `schemas.py` 了解字段；
3. 打开 `service.py` 阅读方法 docstring 的处理顺序；
4. 打开目录 `README.md` 理解上下游调用关系；
5. 打开《接口设计与实现规范》确认事务、幂等、错误和安全规则；
6. 最后查看《系统设计文档》的数据表与伪代码；
7. 只在相应方法体中实现，不改变签名；若签名不合理，先提交 CodeStructureChangeRequest。
