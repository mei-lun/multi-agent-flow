# Multi Agent Flow 开发任务库

每个任务尽量控制在 0.5～2 个开发日，只解决一个主要问题。开发前必须先完成“前置条件”，开发后逐项满足“验收标准”。任务状态由后续 Git coordination 协议维护；本目录当前是实施基线。

## 执行规则

1. 按任务 ID 和依赖推进，不因目录顺序跳过前置条件；
2. 一个任务使用一个分支和一个 PR；
3. 不在当前任务中顺手实现“不包含”内容；
4. 公共接口需要变化时先修改设计和接口契约；
5. 每个验收项都应有测试、命令输出或文档证据；
6. 跨机器协作只使用 Git control/node/task 分支，不新增节点 HTTP。
