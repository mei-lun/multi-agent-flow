# 任务范围清单

本清单与各 TASK 文档绑定。每一行规定该任务可以修改的生产文件、接口/类型、测试文件和禁止边界。除任务文档本身外，未列出的文件默认不可修改。

## 路径表示规则

清单中的缩写是固定展开，不是“任意目录”：

- `apps/server/.../x` 固定表示 `apps/server/src/maf_server/x`；
- `apps/runner/.../x` 固定表示 `apps/runner/src/maf_runner/x`；
- `packages/contracts_py/.../x` 固定表示 `packages/contracts_py/src/maf_contracts/x`；
- `packages/domain/.../x` 固定表示 `packages/domain/src/maf_domain/x`；
- 以 `modules/`、`core/`、`gateway/`、`scheduler/` 开头的路径固定相对于 `apps/server/src/maf_server/`；
- 以 `workspace/`、`docker/`、`runtime/` 开头的路径固定相对于 `apps/runner/src/maf_runner/`；
- `**` 只表示列出目录内部与本任务直接相关的文件，不能扩展到相邻模块；
- 列出的文件当前不存在时允许按该精确路径新增；父目录之外的文件仍禁止新增；
- 测试范围与生产范围分开，不能通过“写测试”修改未授权生产文件。

## 范围变更流程

实现过程中如果必须修改未授权文件或接口，开发者必须停止当前实现并提交 `CodeStructureChangeRequest`，至少写明：

- 当前 `task_id`；
- 希望增加的精确文件路径；
- 希望新增或改变的完整接口签名；
- 原范围无法完成的原因；
- 受影响的调用方、数据、测试和文档；
- 是否兼容旧接口及迁移办法；
- 建议由开发专家、架构师或产品负责人中的谁批准。

批准后先修改对应 TASK 文档和本清单，再修改代码。未获批准不得通过复制类型、跨层导入、动态调用或临时脚本绕过范围。
